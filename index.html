The 7AAST framework

Seven conditions every deployed
clinical AI model must survive.

Every evaluation runs on a synthetic patient population calibrated to federal benchmarks — CDC BRFSS, ACS, IHS. Zero PHI. Fully deterministic. Any third party can reproduce the run independently.

Axis 01
Missingness
5–20% of clinical fields systematically removed. Labs absent. Vitals unrecorded. History incomplete.
Axis 02
Gaussian noise
Continuous variables corrupted within clinically realistic bounds. Sensor drift. Transcription variance.
Axis 03
Temporal drift
18-month deployment simulation. Population characteristics shift over time. The model you trained is not the model you deployed.
Axis 04
Population shift
Out-of-distribution demographic profiles. Rural, tribal, dual-eligible. Patients no benchmark dataset was built around.
Axis 05
Label noise
Systematic inconsistency injected into ground truth coding. Reflects real-world ICD documentation variance.
Axis 06
Semantic perturbation
Identical clinical conditions documented in structurally different ways. Note style, abbreviation, phrasing variation.
Axis 07 New
Representational polymorphism
Two patients. Identical clinical profiles — same age, diagnosis, labs, vitals. One record arrives as a structured FHIR bundle. The other as a free-text clinical note. A reliable model produces consistent recommendations for both. An inconsistent model diverges — not because the clinical picture changed, but because the data representation did. This is the failure mode static benchmarks never surface. It causes unexplained output variance across sites, EHR vendors, and documentation cultures.

Cryptographic artifact pipeline — CAP

Every report is cryptographically sealed.
Any regulator can verify it independently.

Four stages. Each seals a component of the evaluation to everything that preceded it. The final artifact is anchored to the Bitcoin blockchain. No access to HipAAsynth systems required. No trust required — just math.

S1
Population seal
The synthetic patient population is SHA-256 hashed in canonical form. A permanent fingerprint of exactly which patients participated.
SHA-256( patients.json ) → population_hash
S2
Preregistration seal
The study design is hashed together with the population seal. Proves the hypotheses were locked before any model outputs were observed.
SHA-256( prereg.md + S1 ) → prereg_hash
S3
Results seal
Every model output — patient by patient, axis by axis — is canonically sorted and hashed with both previous seals. Results are permanently bound to the population and study design.
SHA-256( results.csv + S2 + S1 ) → results_hash
S4
Final artifact + QR code
All three seals combine into a final artifact hash. A QR code is embedded on the last page of the report. The hash is anchored to the Bitcoin blockchain via OpenTimestamps — permanent, court-admissible, verifiable by anyone.
SHA-256( S3 + S2 + S1 ) → artifact_hash → ⛓ Bitcoin
