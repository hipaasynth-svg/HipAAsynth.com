How It Works

Four steps from seed to dataset. Every run is deterministic and auditable.

1
Choose a Domain
Cardiology, diabetes, rare disease, clinical trials, or general population
2
Set a Seed
Pick any integer. Same seed always produces the same patients, byte for byte
3
Configure
Cohort size, age range, sex ratio, ethnicity weights, visit count, conditions
4
Export
Get CSV, JSON, or FHIR R5 bundles with anchor hash for verification

Sample Output

Every patient record includes demographics, anthropometrics, conditions, visits, and labs.

// seed=42, patient_count=1 — this exact output regenerates every time { "demographics": { "patient_id": "SYN-a1b2c3d4", "seed": 42, "age": 58, "sex": "female", "ethnicity": "white" }, "anthropometrics": { "height_cm": 164.3, "weight_kg": 78.1, "bmi": 28.9, "bmi_category": "overweight" }, "conditions": [ { "name": "hypertension", "onset_age": 44, "active": true }, { "name": "hyperlipidemia", "onset_age": 51, "active": true } ], "visits": [ { "visit_id": "VIS-00a1", "visit_type": "outpatient", "visit_date": "2025-03-12", "labs": [ { "lab_name": "LDL", "value": 142.0, "unit": "mg/dL" } ] } ], "synthetic": true, "engine_version": "1.0.0" }

REST API

Generate cohorts programmatically. One endpoint, deterministic output.

Health Check
GET/health
Generate Cohort
POST/generate
  • JSON or CSV response format
  • Seed parameter for deterministic output
  • Population profile selection
  • Cohort size up to 10,000 per request
  • Summary statistics included in response
  • Anchor hash for verification
// POST /generate { "n_patients": 500, "seed": 42, "format": "json", "profile": "profiles/us_default.json" } // Response includes: { "metadata": { "anchor_hash": "a7f3...", "rows": 500, "deterministic": true }, "summary": { ... }, "records": [ ... ] }
