How It Works

Four steps from seed to dataset. Every run is deterministic and auditable.

1
Choose a Domain
Cardiology, diabetes, rare disease, clinical trials, or general population
2
Set a Seed
Pick any integer. Same seed always produces the same patients, byte for byte
3
Configure
Cohort size, age range, sex ratio, ethnicity weights, visit count, conditions
4
Export
Get CSV, JSON, or FHIR R5 bundles with anchor hash for verification

Sample Patient Record

Every patient record includes demographics, anthropometrics, conditions, visits, and labs.

// seed=42 — same seed always produces the same cohort { "patient_id": "SYN-a1b2c3d4", "age": 58, "sex": "female", "conditions": [ "hypertension", "hyperlipidemia" ], "synthetic": true, // ... demographics, anthropometrics, visits, labs included }
